# プロジェクト概要
留学中の思い出の写真をまとめたZINE（小冊子）を拡張するWebARプロトタイプを作成します。
ユーザーがスマホのブラウザでカメラを起動し、ZINEに印刷された特定の写真（マーカー）にかざすと、その写真の上にぴったりと重なる形で、当時の動画が音声付きで再生されるインタラクションを実装してください。

# 技術スタック
- フロントエンド: HTML5, JavaScript (Vanilla)
- ARフレームワーク: MindAR.js (A-Frame版)
- 3D/VRライブラリ: A-Frame
- 動作環境: iOS Safari, Android Chrome (モバイルブラウザ推奨)

# ディレクトリ・ファイル構成（予定）
/zine
 ├── index.html       // メインの実装ファイル
 ├── target
    └──target1.mind     // コンパイル済みのARマーカーファイル（※ユーザーが用意します）
 ├── pic
    └──picture1.jpeg　　// コンパイル前の画像ファイル（※ユーザーが用意します）
 └── mov
    └──movie1.mp4       // 再生する動画ファイル（※ユーザーが用意します）

# コア機能と要件（厳守事項）
1. Image Tracking (画像認識)
   - MindARの `<a-scene mindar-image="imageTargetSrc: ./targets.mind;">` を使用すること。
   - 写真がカメラに認識されたら（`targetFound`）、動画を表示・再生する。
   - 写真がカメラから外れたら（`targetLost`）、動画を一時停止（pause）し、非表示にする。

2. 動画のオーバーレイ表示
   - `<a-video>` を使用し、ARマーカー（写真）の上に動画を配置する。
   - 動画のアスペクト比が崩れないように、widthとheightを適切に設定する（仮で4:3の比率 `width="1" height="0.75"` としておくこと）。

3. モバイルブラウザのAutoplayポリシー回避
   - iOS/Androidのブラウザでは、ユーザーのアクションなしに音声付き動画を自動再生できない仕様があります。
   - htmlの `<video>` タグには必ず `playsinline webkit-playsinline muted` を付与すること。
   - マーカーを認識した際（`targetFound` のイベント発火時）に、JavaScriptで `video.muted = false` にして音声をオンにし、`video.play()` を実行するロジックを組むこと。

4. UI/UX
   - 画面上の余計なUI（VRボタン等）はすべて非表示にすること（`vr-mode-ui="enabled: false"` を指定）。
   - エラー発生時（カメラの許可が下りない等）は、コンソールに分かりやすくログを出力すること。
