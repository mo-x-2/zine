# ZINE WebAR

留学中の思い出写真をまとめた **ZINE（小冊子）** を拡張する WebAR プロトタイプです。
スマホのブラウザでカメラを起動し、ZINE に印刷された写真（マーカー）にかざすと、
その写真の上にぴったり重なる形で当時の動画が **音声付き** で再生されます。

## 仕組み

1. ユーザーがスマホのブラウザでページを開き「タップして開始」を押す
2. カメラが起動し、画像認識（Image Tracking）が始まる
3. ZINE に印刷された写真をカメラが認識する（`targetFound`）と、写真の上に動画が表示され、音声付きで再生される
4. 写真がカメラから外れる（`targetLost`）と、動画は一時停止して非表示になる

```
 │    └── target1.mind   // コンパイル済みARマーカー（※自分で用意）
 ├── pic/

 └── mov/
      └── movie1.mp4     // 再生する動画（※自分で用意）
```

## 準備するファイル

以下の 2 つを自分で用意し、決められた場所に置きます。ファイル名もこの通りにすると、`index.html` を編集せずにそのまま動きます。

| ファイル              | 置き場所            | 内容                                      |
| --------------------- | ------------------- | ----------------------------------------- |
| `target1.mind`        | `target/` フォルダ  | 写真を AR で認識させるためのマーカーデータ |
| `movie1.mp4`          | `mov/` フォルダ     | 写真に重ねて再生する動画                   |
| `picture1.jpeg`（任意）| `pic/` フォルダ     | マーカーの元画像（`.mind` 生成用の元データ） |

> ファイル名を変えたい場合は、`index.html` 内の該当箇所（コメントで案内しています）を書き換えてください。

## マーカー（.mind）の作り方

AR で認識させる写真は、そのままでは使えません。MindAR の **Image Targets Compiler** で `.mind` 形式に変換（コンパイル）する必要があります。

### 手順

1. **元画像を用意する**
   - ZINE に印刷する写真を、PC に画像ファイル（JPEG / PNG）として保存します。
   - `pic/picture1.jpeg` として置いておくと管理しやすいです。

2. **コンパイラを開く**
   - ブラウザで公式の Image Targets Compiler を開きます：
     👉 https://hiukim.github.io/mind-ar-js-doc/tools/compile/

3. **画像をアップロードする**
   - 「Upload」エリアに用意した写真をドラッグ＆ドロップします。
   - 複数枚を一度にアップロードすれば、1 つの `.mind` に複数マーカーをまとめられます。

4. **コンパイルする**
   - 「Start」ボタンを押すと変換が始まります。

5. **ダウンロードして配置する**
   - 完了したら「Download」を押して `targets.mind` を保存します。
   - ファイル名を `target1.mind` にリネームし、`target/` フォルダに置きます。

### 良いマーカー写真のコツ

- **特徴（模様・コントラスト）が多い写真**ほど認識精度が上がります（風景・テクスチャの豊富なものが◎）。
- 真っ白・単色・ぼやけた写真や、左右対称すぎる写真は認識しづらいです。
- 印刷した実物の写真と、コンパイルに使う画像は**同じもの**を使ってください。
- 印刷サイズが大きいほど認識が安定します。

## 動画ファイルの準備

- 形式は **MP4（H.264 + AAC音声）** を推奨します（iOS / Android の両方で再生しやすいため）。
- ファイルサイズが大きいと読み込みに時間がかかるので、スマホ視聴向けに圧縮しておくと快適です。
- `mov/movie1.mp4` として置きます。
- 動画のアスペクト比は初期設定で **4:3**（`width="1" height="0.75"`）です。
  16:9 など比率が違う動画を使う場合は、[カスタマイズ](#カスタマイズ) を参照してください。

## 使い方（ローカルでのテスト）

> ⚠️ **重要：HTTPS が必須です**
> スマホのブラウザは、カメラを使うサイトに **HTTPS（または `localhost`）** を要求します。
> `http://` のままだとカメラが起動しないので注意してください。

### 方法 A：VSCode の Live Server ＋ ngrok（おすすめ）

1. VSCode に拡張機能 **「Live Server」** をインストールします。
2. `index.html` を右クリック →「**Open with Live Server**」を選択（例：`http://127.0.0.1:5500`）。
3. 別のターミナルで [ngrok](https://ngrok.com/) を起動して HTTPS 化します：
   ```bash
   ngrok http 5500
   ```
4. 表示された `https://xxxx.ngrok-free.app` の URL に、**スマホのブラウザ**からアクセスします。
5. カメラの使用許可を「許可」し、「タップして開始」を押します。

### 方法 B：簡易 HTTP サーバーで起動

ターミナルでこのフォルダに移動して、以下のいずれかを実行します。

```bash
# Python がある場合
python3 -m http.server 5500

# Node.js がある場合
npx serve -l 5500
```

その後、上記と同様に ngrok などで HTTPS 化してスマホからアクセスします。

## カスタマイズ

`index.html` 内のコメントで該当箇所を案内しています。主な変更点は次のとおりです。

### マーカーファイルを変更する

```html
<!-- index.html の <a-scene> -->
mindar-image="imageTargetSrc: ./target/target1.mind; ..."
```

`./target/target1.mind` を任意のパスに変更します。

### 動画ファイルを変更する

```html
<!-- index.html の <a-assets> 内 -->
<video id="zineVideo" src="./mov/movie1.mp4" ...></video>
```

`./mov/movie1.mp4` を任意のパスに変更します。

### 動画の比率（アスペクト比）を変更する

```html
<!-- index.html の <a-video> -->
<a-video id="videoPlane" src="#zineVideo" width="1" height="0.75" ...></a-video>
```

| 比率  | width | height   |
| ----- | ----- | -------- |
| 4:3   | 1     | 0.75     |
| 16:9  | 1     | 0.5625   |
| 1:1   | 1     | 1        |
| 9:16（縦） | 1 | 1.7778   |

### 動画をループさせたくない

`<video>` タグから `loop` 属性を削除してください。

## トラブルシューティング

| 症状                                   | 原因・対処                                                                 |
| -------------------------------------- | -------------------------------------------------------------------------- |
| カメラが起動しない                     | URL が `https://` か確認。`http://` ではスマホのカメラは使えません。       |
| カメラ許可のダイアログが出ない         | ブラウザのサイト設定からカメラを「許可」に変更してください。               |
| 写真を認識しない                       | マーカーの特徴が少ない可能性。コントラストの高い写真で再コンパイルを。     |
| 動画が表示されない                     | `mov/movie1.mp4` のパス・ファイル名が正しいか確認。MP4(H.264) 推奨。       |
| 音が出ない                             | 端末がマナーモード／消音になっていないか確認。初回はタップ操作が必要です。 |
| ライブラリが読み込めない               | インターネット接続を確認（CDN から読み込んでいます）。                     |

エラーの詳細はブラウザの**コンソール（開発者ツール）**に日本語で出力されます。
問題が起きたらまずコンソールを確認してください。
